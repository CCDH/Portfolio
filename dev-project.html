Development

Semester 4 en maatwerk

Mijn maatwerk begon een beetje stroef. Ik wist niet zo goed waar ik moest beginnen. Het idee was om voor mijn development een nieuw portfolio te gaan programmeren die ik ook in semester 6 kon gaan gebruiken. Met als achterliggende gedachte dat ik me in semester 6 geen zorgen hoef te maken over het design van mijn portfolio en hier gewoon mijn werk in kon gaan stoppen.

Om dit portfolio te gaan maken wil ik gaan werken met een framework. Ik heb ervoor gekozen om gebruik te gaan maken van Bootstrap 3. Dit maakt het maken van een website die responsive moet zijn op een hoop devices een stuk makkelijker. In het begin heb ik geprobeerd de beta van Bootstrap 4 te gebruiken. Echter zijn er een aantal dingen veranderd bij deze versie wat resulteerde in een stroeve start van mijn portfolio.

Ook zijn er nog niet zo veel voorbeelden van Bootstrap 4 en is er nog weinig documentatie te vinden. Dit kan er simpelweg voor gaan zorgen dat ik niet de juiste middelen heb om de website te maken. Ook ben ik al een beetje bekend met Bootstrap 3.

Het mooie van Bootstrap is dat het ‘Mobile First’ is opgezet. Dit betekent dat het in de basis uitgaat van een ontwikkeling op een mobiel apparaat zoals een tablet of smartphone.

Waarom Bootstrap?
- In Bootstrap zijn alle CSS-elementen al gemaakt, wat ervoor zorgt dat je zelf geen tijd verliest aan het maken van een CSS-bestand met de juise styling (mits je geen dingen custom wilt maken, dan moet je wel zelf nog een CSS erbij maken)
- Vrijwel alle elementen bestaan in Bootstrap, zoals tabellen, knoppen, formulieren, labels etc.
- Bootstrap wordt op vrijwel elke browser ondersteund
- Bootstrap is geoptimaliseerd voor zowel desktop, als tablets en mobiele telefoons
- Duidelijke documentatie waardoor je, ook wanneer je minder kennis van Bootstrap hebt, snel aan de slag kunt om je website te bouwen.

Omdat ik in semester 4 te weinig code heb laten zien moet er in dit portfolio dus meer code komen te staan. Het gaat hier voornamelijk om Javascript. Natuurlijk mag er ook op een andere manier aangetoond worden dat ik kennis over development beschik.

Tijdens mijn stage heb ik de mogelijkheid gekregen om mijn development maatwerk te doen. Dit kon ik doen door middel van een opdracht die ik en een collega stagiair hebben gekregen. Dit was om de eerste stappen te zetten naar het programmeren van de nieuwe website die ze in de toekomst willen gaan hanteren.

Helaas kwam ik aan het eind van mijn stage een beetje in de knel waardoor ik ervoor gekozen heb om de focus te gaan leggen op het behalen van mijn stage en gewoon het maatwerktraject in te gaan. Echter vind ik het wel relevant om hetgeen dat ik en mijn collega stagiair gemaakt hebben in mijn huidige portfolio te zetten. Dit hebben we gedaan door de bestaande designs die in een eerder stadium al gemaakt waren om te zetten in code. Dit project is te vinden op de homepage van mijn portfolio.

Ik heb eerst gekeken naar welke frameworks er zijn op het gebied van front-end. Aan de hand van dit lijstje ben ik gaan kijken welke het beste bij mij zou passen en waar ik me meer in zou willen verdiepen gedurende mijn maatwerktraject. Luuk gaf wel aan om te kijken naar mijn eigen vaardigheden en hetgeen dat ik wil gaan maken aanpas op deze vaardigheden.

Dit lijstje is als volgt: Angular.JS, React.JS, Vue.JS en Riot.JS.

Ik ben begonnen met het kijken naar Angular omdat deze mij op de een of andere manier het meest aansprak. Ik ben me gaan verdiepen door middel van verschillende tutorials te volgen op YouTube en heb de tutorial gevolgd die op de website stond van Angular.JS. Hier kwam ik echter niet uit omdat deze naar mijn mening best lastig was en ik gewoon nog niet voldoende kennis beschikte over Angular. Ik heb toen via de hulpwebsite we van school hebben gekregen op www.lynda.com een cursus Angular gevolgd.

Gedurende dit proces kwam ik erachter dat ik nog niet voldoende kennis van Javascript had om echt iets goeds neer te zetten met het Angular framework. Ik heb besloten hier niet mee verder te gaan en me bezig te gaan houden met de basis van Javascript.

Dit heb ik op dezelfde manier gedaan als met Angular. Ik ben eerst tutorials gaan kijken op YouTube en heb deze mee gedaan in mijn eigen code editor. Ook heb ik op www.lynda.com grotendeels de cursus gedaan die over Javascript ging. Het account dat we hebben gekregen om in te loggen is tijdens het maatwerktraject gestopt met werken. Ik ben verder gegaan met het volgen van verschillende tutorials en heb hierbij mee gedaan.

Zo heb ik met behulp van een stap voor stap tutorial op Github een breakout game gemaakt met behulp van enkel Javascript. Deze is inclusief comments te vinden op de homepage van mijn portfolio.

Het meedoen met een tutorial is voor mij een prettige manier van leren echter is het niet de meest ideale manier. Dit is toch echt het zelf doen en het zelf maken van dingen. Dit ben ik gaan doen door te gaan kijken naar het maken van een eigen library. Dit was een van de criteria uit semester 4 waar ik nog een onvoldoende voor had.

Ik ben gaan kijken naar het bounce voorbeeld van Luuk die op Github staat. Aan de hand van dit voorbeeld, eigen code en kleine snippets van internet is het me gelukt om een eigen library te maken die aan te passen is via de HTML. Het gaat hier om een kleine library die in mijn geval een SVG-vorm kan roteren, de transparantheid en de grootte kunt aanpassen. De code inclusief comments en documentatie over hoe je deze kunt hergebruiken is te vinden op mijn Github. Deze is te vinden door op het project te klikken op de homepage van mijn portfolio.

Hierna ben ik gaan zitten met Luuk om deze library te laten zien en te kijken wat ik verder nog zelf kon maken met Javascript. Hieruit kwam het maken van een steen, papier en schaar game. Deze hoeft dan niet voorzien te zijn van een mooi design maar enkel zelfgeschreven Javascript code waarmee je het spelletje kunt spelen. In mijn geval is dit door middel van console log. Dit project is ook te vinden op de homepage van mijn portfolio.
